3.2 Overview

The app now includes a dedicated Coverage Calculator tab for logged-in users. It is intended as a planning tool for sequencing design and lets users estimate:

  • the coverage they can expect from a known number of reads or read pairs
  • the number of reads or read pairs they need to reach a desired coverage

The calculator is implemented in sequencing-app/app.R and is available to:

  • regular users: as a second top-level tab next to Apply for Projects
  • administrators: as a new tab alongside Home, Apply for Projects, and Administer Projects

3.3 Where the calculator lives

The calculator is rendered as a 50/50 split layout:

  • left side: inputs and result card
  • right side: definitions, formulas, and usage notes

On smaller screens, the layout stacks vertically so the calculator remains usable on narrow displays.

3.4 What the calculator can do

The first decision in the calculator is the calculation direction:

  1. Coverage from available reads
    • user enters the number of reads / clusters or read pairs / clusters available
    • app estimates the resulting coverage
  2. Required reads for target coverage
    • user enters the desired coverage
    • app estimates the required read count

The UI wording changes automatically so the output matches the chosen sequencing layout.

3.5 Reference genomes and genome sizes

The calculator uses the existing reference_genomes table.

New database support was added:

  • new column: reference_genomes.genome_size_bp
  • runtime migration: the app adds the column automatically when missing
  • seed values: known organisms now receive stored genome sizes in base pairs

Known genomes currently seeded:

  • Caenorhabditis elegans -> 100.3 Mb
  • Danio rerio - Zebrafish -> 1.4 Gb
  • Drosophila melanogaster -> 143.7 Mb
  • Drosophila pseudoobscura -> 163.3 Mb
  • Escherichia coli -> 4.6 Mb
  • Gallus gallus -> 1.1 Gb
  • Homo sapiens -> 3.1 Gb
  • Mus musculus -> 2.7 Gb
  • Saccharomyces cerevisiae - yeast -> 12.1 Mb

Legacy short names were also normalized at startup so older local databases such as:

  • Hsp.GRCh38
  • Mmu.GrCm38
  • Cel.WBcel235

are rewritten to the current full names and keep the correct genome size.

If a selected entry has no stored genome size, the calculator exposes a manual input with units:

  • bp
  • Mb
  • Gb

This is used for entries such as:

  • Custom
  • Others or Mixed

3.6 Read layout modes

The calculator now supports three read-layout modes:

3.6.1 Paired-end (same length for both reads)

Use this when both reads have the same length, for example 2x150.

User input:

  • one read-length value

Internal base calculation:

total bases per pair = 2 x read length

3.6.2 Single-end

Use this when only one read is generated per cluster.

User input:

  • one read-length value

Internal base calculation:

total bases per read = read length

3.6.3 Paired-end (custom Read 1 / Read 2)

Use this when Read 1 and Read 2 differ.

User input:

  • Read 1 length
  • Read 2 length

Internal base calculation:

total bases per pair = Read 1 length + Read 2 length

3.7 Read-count input units

Read-count inputs are now entered in millions.

Examples:

  • 200 means 200,000,000
  • 310 means 310,000,000
  • 0.5 means 500,000

The result card displays:

  • the primary result in millions
  • the exact absolute count as a supporting line below

This keeps large sequencing numbers easier to read without losing precision.

3.8 Formulas used

3.8.1 Coverage from available reads

Single-end:

coverage = (reads / clusters) x (read length) / genome size

Paired-end, same length:

coverage = (read pairs / clusters) x (2 x read length) / genome size

Paired-end, custom:

coverage = (read pairs / clusters) x (Read 1 length + Read 2 length) / genome size

3.8.2 Required reads for target coverage

Single-end:

required reads / clusters = (desired coverage x genome size) / read length

Paired-end, same length:

required read pairs / clusters = (desired coverage x genome size) / (2 x read length)

Paired-end, custom:

required read pairs / clusters = (desired coverage x genome size) / (Read 1 length + Read 2 length)

The app rounds required counts up with ceiling(...) so the returned target is not underestimated.

3.9 Admin maintenance

Genome sizes can now be managed in the admin UI:

  1. Open Administer Projects
  2. Click Manage Genomes
  3. Add or edit:
    • genome name
    • optional genome size
    • unit interpretation is stored internally as base pairs

If the size is left empty, the genome remains valid in the dropdown but requires manual size entry in the calculator.

3.10 Validation checklist

Recommended test cases after deploy:

  1. Select Homo sapiens, choose paired-end same length, enter 150, target 30x:
    • result should be about 310 million
  2. Select Homo sapiens, paired-end same length, available reads 310:
    • result should be about 30x
  3. Select Escherichia coli, paired-end same length, available reads 10, read length 150:
    • result should be about 652.2x
  4. Select Custom:
    • manual genome-size input should appear
  5. Switch between single-end and paired-end modes:
    • read-length fields and labels should update automatically
